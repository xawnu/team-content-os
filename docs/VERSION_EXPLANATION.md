# Team Content OS — 版本介绍（PRD v1.0）

## 1. 产品目标
为小团队提供一套从“对标研究 → 选题/文案生成 → 执行复盘”的闭环系统，提升内容生产效率与稳定性，减少拍脑袋选题和低质量脚本。

## 2. 目标用户
- 3-20 人内容团队（YouTube/短视频方向）
- 核心角色：内容负责人、策划、编导、数据复盘人员

## 3. 核心问题
1. 对标分析分散，频道/视频信息难沉淀
2. 文案生成容易跑偏、空泛、重复
3. 历史产出不可追踪，复盘链路断裂
4. 多页面导航不统一，操作成本高

## 4. 产品范围（当前）
### 4.1 增长频道发现（/discover）
- 关键词 + 赛道模板抓取候选频道
- 展示候选分数、样例标题、历史批次
- 支持“加入参考视频池”

### 4.2 种子找同类（/similar）
- 输入 seed channel（@handle/链接）找同类频道
- 历史记录与详情回看

### 4.3 文案生成器（/planner）
- 仅支持 1 篇详细文案（先做质量）
- 必须基于“参考视频池”生成（无参考不生成）
- 可选约束：主题锁定、禁用词、内容风格
- 强质量校验（数量承诺一致、分镜段数/长度/镜头动作等）
- 生成结果自动落库到历史
- 历史支持查看详情与删除

### 4.4 追踪复盘（/tracker）
- 回填指标：CTR、30s留存、7日播放等
- 输出基础策略建议

### 4.5 自动周报（/reports）
- 聚合近 7 天表现，输出 winners/losers 与下周建议

### 4.6 统一导航
- 顶部全局导航：Dashboard / 发现 / 同类 / 文案 / 追踪 / 周报 / 版本介绍

## 5. 关键功能需求（FR）
- FR-01 固定邮箱+密码白名单登录
- FR-02 对标发现与历史查询
- FR-03 参考视频池（1-3条）驱动文案生成
- FR-04 文案质量校验，不达标禁止返回
- FR-05 文案历史持久化
- FR-06 历史详情查看（分镜、要点、参考视频）
- FR-07 历史删除
- FR-08 统一导航与 Dashboard 指标展示

## 6. 非功能需求（NFR）
- 响应：常规页面 < 2s，生成任务 10-30s 可接受
- 稳定：异常时明确报错，不静默失败
- 安全：密码不明文入库/入仓
- 可维护：关键流程可追踪

## 7. 当前技术方案
- Next.js (App Router)
- PostgreSQL + Prisma
- PM2 部署
- AI 调用：OpenRouter + 本地规则/质量校验兜底

## 8. 验收标准（MVP）
1. 用户可登录并进入系统
2. 可从发现页加入参考视频池
3. Planner 有参考视频时可生成 1 篇详细文案
4. 生成结果可在历史查看详情并删除
5. 顶部导航在主页面一致
6. 质量不达标时有清晰错误提示

## 9. 已知问题 / 待优化
- 模型波动仍会影响生成质量
- 历史详情对旧记录字段不完整
- 参考视频池目前为文本态，建议升级为可视化卡片
- 可新增“再来一版”快速变体

## 10. 下一阶段（v1.1）
1. 参考视频池可视化（拖拽排序/删除）
2. 一键“再来一版”（同约束不同文案）
3. 结果评分机制（结构完整度、可拍摄性、去空话）
4. 按频道类型沉淀高质量模板库
